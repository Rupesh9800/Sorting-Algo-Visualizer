Quick Sort

Definition: Quick Sort is an efficient and widely used comparison-based sorting algorithm. It employs a divide-and-conquer strategy to sort elements by partitioning the array into smaller sub-arrays.

How It Works:

  1. Choose a Pivot: Select an element from the array to serve as the pivot.
  2. Partitioning: Rearrange the elements so that elements less than the pivot are on the left, and elements greater than the pivot are on the right.
  3. Recursively Sort: Recursively apply the same process to the left and right sub-arrays until the entire array is sorted.

Steps Example (Ascending Order):

Given an unsorted array: [10, 7, 8, 9, 1, 5]

  • Step 1: Choose a pivot (e.g., 5).
    Partition the array around the pivot.
    Result: [1, 5, 7, 8, 9, 10]
  • Step 2: Recursively apply Quick Sort to the left sub-array [1] (already sorted).
  • Step 3: Recursively apply Quick Sort to the right sub-array [7, 8, 9, 10].
    Choose a pivot (e.g., 9).
    Partition the array around the pivot.
    Result: [7, 8, 9, 10]

Characteristics:

  • Time Complexity: O(n²) in the worst case (when poorly chosen pivots lead to unbalanced partitions), O(n log n) on average.
  • Space Complexity: O(log n) due to recursive calls (in-place sorting).
  • Stability: Not stable (does not preserve the relative order of equal elements).
  • Use Case: Suitable for large datasets and commonly used due to its average-case efficiency.

Advantages:

  • Generally faster than other O(n log n) algorithms like Merge Sort and Heap Sort for large datasets.
  • In-place sorting requires less additional memory compared to Merge Sort.

Disadvantages:

  • Worst-case time complexity of O(n²), although this can be mitigated with good pivot selection strategies (e.g., random pivot, median-of-three).
  • Not stable, meaning equal elements might not retain their relative order.
