Merge Sort

Definition: Merge Sort is a stable, comparison-based sorting algorithm that follows the divide-and-conquer approach. It divides the array into smaller sub-arrays, sorts them recursively, and then merges them to produce a sorted array.

How It Works:

  1. Divide: Divide the unsorted array into two halves recursively until each sub-array contains only one element.
  2. Conquer: Merge the smaller sorted arrays (left and right halves) to create larger sorted arrays until the entire array is merged and sorted.

Steps Example (Ascending Order):

Given an unsorted array: [38, 27, 43, 3, 9, 82, 10]

  • Step 1: Divide the array into smaller sub-arrays until each sub-array contains one element.
    Recursive splitting: [38, 27, 43, 3, 9, 82, 10] → [38, 27, 43, 3], [9, 82, 10]
  • Step 2: Merge the smaller sorted arrays back together in sorted order.
    First merge: [27, 38, 43, 3] and [9, 10, 82] → [3, 9, 10, 27, 38, 43, 82]

Characteristics:

  • Time Complexity: O(n log n) in all cases (best, average, and worst).
  • Space Complexity: O(n) due to the auxiliary space required for merging.
  • Stability: Stable (preserves the relative order of equal elements).
  • Use Case: Suitable for sorting linked lists and external sorting where random access is costly.

Advantages:

  • Efficient and consistent performance across different input types.
  • Stable sorting algorithm suitable for scenarios where maintaining order is crucial.

Disadvantages:

  • Requires additional memory space for merging sub-arrays, which can be a drawback for large datasets.
  • Not as efficient in practice as Quick Sort for arrays with a small size or when a high level of memory is available.
