Insertion Sort

Definition: Insertion Sort is a simple and intuitive comparison-based sorting algorithm. It builds the sorted array one item at a time, gradually creating a larger left-hand portion which is always sorted.

How It Works:

  1. Initial Setup: The algorithm starts with the first element, considering it as sorted.
  2. Iterate and Insert: For each subsequent element, it compares it with elements in the sorted portion (left side) and inserts it into the correct position by shifting the other elements as necessary.
  3. Repeat: Repeat the process for all elements in the array.

Steps Example (Ascending Order):

Given an unsorted array: [12, 11, 13, 5, 6]

  • Step 1: The first element (12) is already sorted.
    Array: [12, 11, 13, 5, 6]
  • Step 2: Take the next element (11) and insert it into the sorted portion.
    Array: [11, 12, 13, 5, 6]
  • Step 3: Take the next element (13) and insert it into the sorted portion.
    Array: [11, 12, 13, 5, 6]
  • Step 4: Take the next element (5) and insert it into the sorted portion.
    Array: [5, 11, 12, 13, 6]
  • Step 5: Take the last element (6) and insert it into the sorted portion.
    Final sorted array: [5, 6, 11, 12, 13]

Characteristics:

  • Time Complexity: O(nÂ²) in the average and worst case, O(n) in the best case (when the array is already sorted).
  • Space Complexity: O(1) (in-place sorting).
  • Stability: Stable (preserves the relative order of equal elements).
  • Use Case: Suitable for small datasets or nearly sorted datasets.

Advantages:

  • Simple to implement and understand.
  • Efficient for small or nearly sorted datasets.
  • Adaptive: Efficient for data sets that are already substantially sorted.

Disadvantages:

  • Inefficient for large datasets compared to more advanced algorithms like Quick Sort, Merge Sort, and Heap Sort.
  • The number of comparisons and shifts can be high for large or random datasets.
