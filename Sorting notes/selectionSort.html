Selection Sort

Definition: Selection Sort is a simple comparison-based sorting algorithm that divides the input list into two parts: the sublist of items already sorted, which is built up from left to right at the front (left) of the list, and the sublist of items remaining to be sorted that occupy the rest of the list.

How It Works:

  1. Initial Setup: The algorithm starts with the entire list as unsorted.
  2. Finding the Minimum: From the unsorted sublist, find the smallest element.
  3. Swapping: Swap this smallest element with the first element of the unsorted sublist.
  4. Move Boundary: Move the boundary between the sorted and unsorted sublist one element to the right.
  5. Repeat: Repeat the process for the remaining unsorted elements.

Example:

Given an unsorted array: [64, 25, 12, 22, 11]

  • Step 1: Find the minimum element (11) and swap it with the first element (64).
    Array after step 1: [11, 25, 12, 22, 64]
  • Step 2: Find the next minimum element (12) and swap it with the second element (25).
    Array after step 2: [11, 12, 25, 22, 64]
  • Step 3: Find the next minimum element (22) and swap it with the third element (25).
    Array after step 3: [11, 12, 22, 25, 64]
  • Step 4: The last two elements (25 and 64) are already in order.
    Final sorted array: [11, 12, 22, 25, 64]

Characteristics:

  • Time Complexity: O(nÂ²) in all cases (best, average, and worst).
  • Space Complexity: O(1) (in-place sorting).
  • Stability: Not stable (does not preserve the relative order of equal elements).
  • Use Case: Suitable for small datasets or when memory space is limited.

Advantages:

  • Simple to implement and understand.
  • Works well with small lists.

Disadvantages:

  • Inefficient on large lists compared to more advanced algorithms like Quick Sort, Merge Sort, and Heap Sort.
  • The number of comparisons grows quadratically with the number of elements.
